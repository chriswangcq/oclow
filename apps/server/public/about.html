AI Meditations

AI Meditations is a private Markdown knowledge workspace. Humans read it like a lightweight wiki; agents access the same files through MCP and help compile raw material into a stable, traceable LLM Wiki.

What it does

  • sources/ keeps raw material and provenance.
  • docs/ keeps compiled long-term knowledge.
  • journal/ keeps sessions, changes, and daily timelines.
  • self/ keeps user-visible, durable agent context.

Inside docs/, direct docs/<slug>/ folders are top-level documents. Each folder is a document package: README.md is the body, sibling Markdown files are pages, sub_docs/ contains child documents, and _attachments/ stores files owned by the current document.

How Google sign-in is used

Google sign-in is used only to identify your account, create a session, and isolate your private workspace. AI Meditations requests only basic profile information: Google account ID, email, name, and avatar when provided. It does not request Gmail, Google Drive, Google Calendar, contacts, or other Google product data.

Agent access

AI Meditations provides Remote MCP access. After you authorize a client, an agent can read and write Markdown files inside your own workspace under explicit file sandbox rules. Your workspace remains private to your account.

中文

AI 沉思录

AI 沉思录是一个私有 Markdown 知识工作区。人可以像阅读轻量 wiki 一样查看长期知识； Agent 可以通过 MCP 读写同一套文件，把原始资料编译成稳定、可追溯的 LLM Wiki。

它做什么

  • 用 sources/ 保存原始资料和来源。
  • 用 docs/ 保存编译后的长期知识。
  • 用 journal/ 记录会话、变更和每日时间线。
  • 用 self/ 保存用户可见、长期稳定的 Agent 上下文。

在 docs/ 里，直接的 docs/<slug>/ 文件夹是顶层文档。 每个文件夹都是文档包：README.md 是正文，同级 Markdown 是页面，sub_docs/ 里放子文档， _attachments/ 存放当前文档的附件。

Google 登录的用途

Google 登录只用于识别账号、创建会话和隔离用户工作区。AI 沉思录只请求基础登录信息： Google 账号 ID、邮箱、姓名和头像。应用不会请求 Gmail、Google Drive、Google Calendar 或其他 Google 产品数据。

Agent 接入

AI 沉思录提供 Remote MCP 接入。用户授权后，Agent 可以在用户自己的工作区中按规则读取和写入 Markdown 文件。所有数据仍属于用户的私有工作区。
